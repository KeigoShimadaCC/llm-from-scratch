# PHASE-10B - Corpus Download And Cleaning

## Goal

Implement reproducible, CI-safe download, extraction, and cleaning commands for the approved `corpus_v01` sources without committing raw or processed corpus artifacts.

## Prerequisites

- PHASE-10A is complete.
- `configs/corpus_v01.yaml` and `docs/corpus_v01_source_manifest.md` exist and pass source audit.
- All included sources have documented license/provenance/attribution metadata.

## Scope

- `corpus.download` CLI with `--dry-run` that validates planned downloads and storage paths without network payload writes.
- `corpus.clean` CLI with `--smoke` that creates a tiny ignored processed corpus from local sample data or bounded local source excerpts.
- Wikipedia extraction/cleaning plan for English and Japanese dumps.
- Project Gutenberg header/footer and boilerplate cleaning plan.
- Aozora Bunko metadata, ruby, gaiji, and annotation cleaning plan.
- Structured processed JSONL format for downstream split and tokenizer phases.

## Allowed Paths

- `configs/**`
- `corpus/**`
- `data/README.md`
- `docs/**`
- `tests/**`
- `PROGRESS.md`

## Forbidden Paths

Forbidden paths are forbidden to commit or manually edit as phase source changes. Required commands may generate ignored evidence artifacts only when this plan's artifact policy allows them.

- `.env`
- `.env.*`
- `data/raw/**`
- `data/processed/**`
- `data/tokenized/**`
- `experiments/runs/**`
- `runs/**`
- `*.pt`
- `*.safetensors`
- `kgpt/**`
- `tokenizer/**`
- `train/**`
- `eval/**`
- `inference/**`
- `automation/**`
- `phase-plans/**`

## Phase Dependencies

- Depends on PHASE-10A.
- Unblocks PHASE-10C split, dedup, leakage, and dataset manifest work.

## Tasks

- Implement `corpus.download --dry-run` to enumerate approved source downloads, target ignored raw paths, expected extraction paths, and blocker status.
- Implement real download code paths guarded by explicit config/source allowlist and resumable file handling, while keeping CI validation dry-run only.
- Implement `corpus.clean --smoke` to write a bounded ignored processed corpus under `data/processed/corpus_v01_smoke`.
- Define processed JSONL records with stable fields: `doc_id`, `source_id`, `lang`, `title`, `text`, `license`, `attribution`, `source_url`, `source_record_id`, `sha256`, and `cleaning_version`.
- Add cleaning routines for:
  - Wikipedia markup/template/ref/table/category noise where feasible.
  - Project Gutenberg legal boilerplate, header/footer markers, catalog metadata, and repeated whitespace.
  - Aozora ruby annotations, gaiji markers, metadata blocks, and repeated whitespace.
  - Unicode normalization, line normalization, minimum length filters, language tags, and empty-text rejection.
- Add smoke fixtures that are repo-authored or tiny quoted metadata-only samples, not corpus payloads.
- Add tests for dry-run download planning, cleaning transforms, JSONL schema, deterministic hashes, and source allowlist rejection.

## Deliverables

- `corpus.download` CLI.
- `corpus.clean` CLI.
- Cleaning helpers and schema validation.
- Tests for download planning, cleaning, and JSONL output.
- Updated `data/README.md` with raw and processed local storage conventions.
- `PROGRESS.md` update with ignored smoke output path.

## Evidence Artifacts

- Ignored `data/processed/corpus_v01_smoke/**` generated by the smoke cleaning command.
- Optional ignored download plan or runner logs under `runs/phase-runner/PHASE-10B/**`.
- Committed docs/test evidence only; no raw or processed corpus text is committed.

## Acceptance Criteria

- Dry-run download validation succeeds without downloading corpus payloads.
- Smoke cleaning produces deterministic processed JSONL records with source/provenance/license/attribution metadata preserved per document.
- Cleaning never silently includes sources outside the PHASE-10A allowlist.
- The phase documents the difference between CI smoke evidence and full local download/clean artifacts.
- Full downloads remain manual or automation-controlled local artifacts, not Git artifacts.

## Required Validation

- `uv run pytest`
- `uv run ruff check .`
- `git diff --check`
- `uv run python -m corpus.download --config configs/corpus_v01.yaml --dry-run`
- `uv run python -m corpus.clean --config configs/corpus_v01.yaml --smoke --output data/processed/corpus_v01_smoke`

## Artifact Policy

- `data/raw/**` and `data/processed/**` are ignored local artifacts and must not be committed.
- Smoke mode may generate ignored processed artifacts required for downstream PHASE-10C validation.
- Committed tests may use repo-authored miniature fixtures only.

## Human Decisions

- Approve running full downloads outside CI if the source audit remains valid.
- Decide whether any cleaning transform is too destructive for educational analysis.
- Approve excluded document classes if license, boilerplate, or quality filters remove large subsets.

## Phase Gate

Mark complete only when dry-run download and smoke cleaning commands pass, processed JSONL schema is tested, ignored artifact policy is verified, and `PROGRESS.md` records how to reproduce the smoke artifacts.

## Risks

- Wikipedia parsing can become complex; smoke validation should prove the local contract without pretending it is a full production extractor.
- Gutenberg and Aozora metadata conventions vary by item and may require conservative exclusion.

## Out Of Scope

- Train/validation/test splitting.
- Tokenizer training or token array generation.
- Model training.
- Storing large corpus artifacts in Git.

## Deferred Backlog

- Add richer quality scoring after the first split/dedup pass exposes corpus composition.
